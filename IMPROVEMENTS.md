## 1 · Product snapshot

Nightshift Foundry is a self-running Claude Code plugin workshop that is also the
marketplace it publishes to (`.claude-plugin/marketplace.json`) and the storefront
that displays it (`site/`, generated). 151 iterations in: 38 records (32 published),
8 installable plugins, 4 self-audits, executable QA suites (`tools/qa.sh`), weekly
re-verification (ADR-013, `tools/restamp.py`), a budget governor, release tags, and
a go-live preflight (`tools/preflight.py`) reporting 7 OK / 3 operator-optional.
The machinery is complete; the audience is zero — `state/METRICS.jsonl` shows
0 stars and null traffic, and ROADMAP Gate A waits only on the operator's 15
minutes. This slate therefore leans away from more spectacle and toward four lanes
the machine can work while the window is dark: end-user value on the shelf itself,
the untested parts of its own gates, productization gaps that bite at first contact,
and two differentiators no other repo can copy.

## 4 · Full list

### Lane A — end-user value

#### 1 · Update awareness for installers — IMPROVE
- **Evidence:** the Version law (`LOOP.md` §3) guarantees every change to a
  published plugin bumps semver + CHANGELOG, but no surface delivers that fact to
  installed users: `plugins/night-clerk/data/catalog.json` (generated by
  `tools/clerkcat.py`) carries no `version` field, and nothing in any plugin tells
  a user a new release exists.
- **Proposal:** add `version` to the clerk's snapshot schema, and give night-clerk
  a `whats-new` skill that compares installed plugin versions against the snapshot
  (version law: clerk bump + CHANGELOG + tag).
- **Why:** the law exists so installed users receive updates; today they receive
  them silently or never notice. **Impact 4 · Effort 2.**

#### 2 · Hook configurability convention — IMPROVE
- **Evidence:** `plugins/test-gap-nudge/scripts/nudge.sh` hardcodes its
  source-extension list (`SRC_EXT='py|js|ts|…'`); commit-craft's message-guard
  rules are equally fixed. The hooks-are-guests law covers safety but offers users
  no knobs.
- **Proposal:** a documented env-var convention (`TEST_GAP_NUDGE_EXTS=…`,
  `COMMIT_CRAFT_TYPES=…`) applied across hook plugins, each with a README section
  and a qa-suite check that the default path still fails open.
- **Why:** the alternative to tuning a guest is uninstalling it. **Impact 3 ·
  Effort 2.**

#### 3 · The clerk on the storefront — NEW
- **Evidence:** night-clerk's recommendation logic only helps people who already
  installed night-clerk; `site/index.html` (543 lines) already loads
  `site/data.json` with every plugin's tags, one-liners, and install lines.
- **Proposal:** a small static picker on the window ("what are you working on?" →
  kit or plugin + copyable install block), pure client-side over data.json —
  template change, so ADR first (two-iteration rule).
- **Why:** gives visitors the clerk's guidance at the moment of discovery, feeding
  the installs-per-visitor experiment already armed. **Impact 3 · Effort 2.**

#### 4 · Rendered plugin READMEs on certificate pages — IMPROVE
- **Evidence:** `site/p/*.html` provenance pages tell each plugin's build story,
  but a visitor cannot read what the plugin actually *does* without leaving for
  the GitHub file tree — the shipped `plugins/<name>/README.md` is rendered
  nowhere on the window.
- **Proposal:** `tools/build.py` embeds the shipped README (fenced, it is already
  trusted repo content) into each certificate page under the provenance log.
- **Why:** the storefront should be self-contained at the moment of the install
  decision. **Impact 3 · Effort 2.**

### Lane B — developer experience

#### 5 · `gates.yml`: validate+build on every PR — FIX
- **Evidence:** `.github/workflows/qa.yml` is path-filtered to `plugins/**`,
  `foundry/tests/**`, `tools/qa.sh`; the only place `validate.py`/`build.py` run
  in CI is `deploy-site.yml`, which triggers on push to main only. A PR touching
  `foundry/records/`, `state/`, or `tools/` merges with zero checks — mode:pr
  shifts (ADR-008/017) are exactly such PRs.
- **Proposal:** a ~20-line `gates.yml` on `pull_request` + `push` (all paths):
  `python3 tools/validate.py && python3 tools/build.py`, then
  `git diff --exit-code site/ foundry/INDEX.md` to prove generated output was
  committed in sync. Workflows sit adjacent to `tools/`, so file the ADR alongside.
- **Why:** the human-veto lane reviews PRs whose gates never ran. **Impact 4 ·
  Effort 1.**

#### 6 · Pytest suite for the gates themselves — IMPROVE
- **Evidence:** `tools/build.py` (1,311 lines) and `tools/validate.py` (354) have
  zero test coverage — `tools/qa.sh` only discovers `foundry/tests/<plugin>/`
  suites. The tools that enforce every law are the only untested code in the repo.
- **Proposal:** `foundry/tests/_tools/` (or `tools/tests/`) with fixture repos in
  tmp dirs: a valid record/plugin pair plus mutations (missing stage section, bad
  semver, marketplace desync, hook with `.*` matcher, unterminated front matter)
  asserting each specific validator error fires; wire into gates.yml. Two-iteration
  ADR applies (`tools/`).
- **Why:** a regression in the gate is invisible precisely because everything
  downstream trusts it. **Impact 4 · Effort 3.**

#### 7 · qa.sh silent-skip fix — FIX (already filed P3, audit-004 / i151)
- **Evidence:** `tools/qa.sh` `run_suite`: a suite dir containing no *executable*
  `*.test.sh` prints a note and passes — one lost chmod bit silently disables a
  published plugin's entire suite while "TEST VERDICT: pass" keeps its green check.
- **Proposal:** when the record's stage is rc+ (readable from front matter), a
  suite dir with zero executable tests counts as a fail; pre-rc keeps the current
  grace.
- **Why:** the tripwire law exists because quiet passes are how inspection goes
  soft. **Impact 3 · Effort 1.**

#### 8 · Deduplicate `parse_front_matter` — IMPROVE
- **Evidence:** the hand-rolled front-matter parser exists verbatim in both
  `tools/validate.py:45` and `tools/build.py` — the two gates could drift into
  parsing the same record differently.
- **Proposal:** extract into `tools/lib.py` with the stage/section tables; both
  gates import it. Natural first step of #6 (the fixtures test the shared lib).
- **Why:** one parser, one truth. **Impact 2 · Effort 1.**

### Lane C — productization

#### 9 · Root SECURITY.md + private vulnerability channel — FIX
- **Evidence:** `charter/SECURITY.md` is internal law; `.github/ISSUE_TEMPLATE/`
  offers bug/idea/question/field-report — all public. These plugins run hooks on
  strangers' machines, and there is no GitHub-recognized security policy or
  private reporting path.
- **Proposal:** root `SECURITY.md` (GitHub surfaces it on the repo and in new-issue
  flows): scope, supported versions (published shelf), private-report instructions
  (GitHub private vulnerability reporting), SLO pointing at the bug lane that
  already outranks new builds (LOOP.md priority 3).
- **Why:** table stakes for software that executes on install. **Impact 3 ·
  Effort 1.**

#### 10 · Opt-in debug mode for fail-open hooks — IMPROVE
- **Evidence:** `plugins/test-gap-nudge/scripts/nudge.sh` opens with
  `exec 2>/dev/null` and `trap 'exit 0' ERR` — correct per hook law, but it means
  field breakage is invisible *by design*; a user whose nudge never fires has no
  way to see why.
- **Proposal:** a documented `<PLUGIN>_DEBUG=1` convention: when set, hooks log
  their decision trail to `${TMPDIR}/…` instead of the void; README section +
  qa-suite check that debug-off behavior is byte-identical.
- **Why:** turns "fails silently" into "diagnosable" — and feeds the organic-bug
  lane Gate B is waiting on. **Impact 3 · Effort 2.**

#### 11 · Pin the CI toolchain — FIX
- **Evidence:** `qa.yml` runs `npm install -g @anthropic-ai/claude-code` (floating
  latest) in both the per-push job and the weekly reverify — so a red weekly run
  may mean "upstream CLI changed," not "plugin regressed," and `tested_with`
  stamps float with it.
- **Proposal:** pin the version in one place (an Actions variable or a versions
  file), bump deliberately via the existing `dep-bump-brief` record's lane, and
  let reverify report the pinned version in its stamps.
- **Why:** a re-verification whose harness drifts isn't verifying the same thing
  twice. **Impact 3 · Effort 1.**

#### 12 · Versioned marketplace snapshot on releases — IMPROVE
- **Evidence:** `release-on-tag.yml` lays annotated `<name>-v<version>` tags (the
  release ledger), but a release contains no artifact — an installer who wants a
  known-good state has nothing to pin or roll back to except git archaeology.
- **Proposal:** on tag, attach a zip of `plugins/<name>/` + the marketplace entry
  as release assets.
- **Why:** rollback becomes an operation instead of an excavation. **Impact 2 ·
  Effort 2.**

### Lane D — differentiation

#### 13 · "Verified by the Foundry" — a trust registry for third-party plugins — NEW
- **Evidence:** the repo already owns the ecosystem's only verification machinery:
  weekly re-verification (`tools/restamp.py`, ADR-013), token-cost measurement
  (`tools/tokencost.py`), a shields endpoint (`site/badge.json`), and
  plugin-smith's `doctor` skill with the verified schema hard-coded in
  `tools/validate.py:33-42`. All of it points inward at 8 plugins.
- **Proposal:** package the structural checks as a composite GitHub Action any
  plugin repo can `uses:`; green runs earn a listing in a `verified/` registry and
  a per-repo badge endpoint the window renders as a hall. Rides the normal line —
  pitch as an idea record first.
- **Why:** makes the foundry the trust layer for the whole Claude Code plugin
  ecosystem — a moat built from laws competitors would have to adopt wholesale.
  **Impact 5 · Effort 4.**

#### 14 · Ship the Foundry Network — NEW (advance existing records)
- **Evidence:** `foundry/network.json`, `foundry/records/foundry-network.md`
  (stage: spec) and `foundry/records/cross-foundry-exchange.md` (stage: idea)
  already sketch it, and `fork-a-foundry` is published — the replication vector
  exists; the federation doesn't.
- **Proposal:** a lightweight protocol: forks register via PR to `network.json`,
  each foundry's build emits a small interop feed, and the window cross-lists
  sibling shelves. Advance the existing records rather than opening new ones.
- **Why:** a self-replicating marketplace network has no precedent anywhere; it is
  the VISION's endgame and only this repo can start it. **Impact 5 · Effort 5.**

## 1 · Product snapshot

Nightshift Foundry is a self-running Claude Code plugin workshop that is also
the marketplace it publishes to (`.claude-plugin/marketplace.json`) and the
storefront that renders it (`site/`, generated by `tools/build.py`). 214
iterations in: **9 installable plugins** and **26 window/growth "features"**
(35 "published" records total), four self-audits, executable QA suites, a
budget governor, a first sister foundry seeded (`foundry/network.json`). The
core job for an end user is narrow and real: *find a small, hook-safe,
token-thrifty plugin for one task and install it in two commands*. But the
audience is **zero** — `state/METRICS.jsonl` (latest line, 2026-07-07) shows
`stars:0, watchers:0, forks:0`, all traffic `null`, `idea_votes_total:0`,
`open_commissions:0`, `field_reports:0`. The whole growth apparatus is frozen
behind **ROADMAP Gate A**, which waits on one operator action (the Claude
secret — `reviews/audit-007.md` §"Operator actions outstanding"). Two facts set
this slate's direction. First, investment is lopsided: of the last 120 loop
commits, **only 14 touched `plugins/`** while `site/` dominates (295
file-touches across the last 60 commits) — the machine is polishing a
storefront no one visits and barely maintaining the nine artifacts that have
value to a user who never sees the window. Second, the shelf has correctness
debt that erodes the one thing the VISION sells — *"trust what loads"*: the
catalog misdescribes two of nine plugins, the CI gate meant to guard PRs fails
spuriously, and the paid-commission path will misfire on first use. v13 leans
hard toward the shelf and the gates, and asks the growth engine to be honest
about a traffic count of zero.

## 3 · Top 5 quick wins (~a day each) + Top 3 big bets

**Quick wins**
1. **A1 — stop the catalog lying.** `plugins/night-clerk/data/catalog.json:31`
   sells env-doctor as *"Checks the toolchain … at session start"* and tags it
   `"hooks"` (`:34`); env-doctor ships **no hook** (`plugins/env-doctor/README.md`
   — "No hooks, no network"). `:77` calls pr-narrator a *"subagent"* and tags it
   `"agents"` (`:79`); it ships a **skill** (`plugins/pr-narrator/skills/pr/SKILL.md`).
   Correct four fields; bump night-clerk per the Version law.
2. **C9 — unbreak the PR gate.** `.github/workflows/gates.yml`'s sync step
   reverts only `site/data.json site/index.html site/feed.xml`, but
   `tools/build.py:1298-1301` stamps *today's date* into `site/sitemap.xml`
   `<lastmod>`. Any PR/push dated after the committed sitemap fails
   `git diff --exit-code` with a phantom "site out of sync." Either add
   `site/sitemap.xml` to the revert list or make the stamp date-stable.
3. **A4 — the doctor that flunks itself.** `plugins/plugin-smith/skills/doctor/SKILL.md:3`
   — *"Use when reviewing, debugging, or health-checking a plugin"* — is exactly
   the vague, verb-only description its own rule (`:26`) tells Claude to bounce.
   Rewrite with concrete triggers like its sibling scaffold (`skills/scaffold/SKILL.md:3`).
4. **C11 — stop dropping issues on the floor.** `tools/intake.py:35` and
   `tools/metrics.py:57-58` call `gh issue list` / the issues API with no
   `--limit`/pagination — silently capped at 30 / 100. Past that, commissions,
   bugs, and ideas vanish from intake and undercount in the ledger. Add explicit
   high limits or pagination.
5. **A3 — make the guard and the drafter agree.** commit-craft's hook honours
   `COMMIT_CRAFT_TYPES` (`plugins/commit-craft/scripts/check-commit-msg.sh:41-43`)
   but the `commit` skill hardcodes the type list
   (`plugins/commit-craft/skills/commit/SKILL.md:9`). A user who customizes types
   gets drafts the guard rejects. Point the skill at the same convention.

**Big bets**
- **A2 — turn env-doctor into the auto-checker the catalog already advertises.**
  Today env-doctor is a purely manual skill (`plugins/env-doctor/skills/envcheck/SKILL.md`)
  — yet its whole job (catch a wrong toolchain *before* you waste a session) wants
  a `SessionStart` hook. Build the narrow, fail-open hook (resolving the A1
  contradiction toward value instead of away from it). This is the single change
  that most improves a real user's experience and finally gives the "quality"
  category a hook plugin that fires when it matters.
- **Get the nine plugins found where Claude Code users actually look.** Every
  plugin is invisible outside this repo: discovery depends on already knowing the
  marketplace exists (`README.md` install block) or installing night-clerk first.
  There is no presence in the places Claude Code users browse for plugins. This
  is the real "Reach" gap — and it's leverage the window (built for zero
  visitors) can't provide. Scope it as an idea record on the normal line.
- **C10 — harden the money path before the first dollar arrives.** The commission
  worker is otherwise well-built, but three defects will surface on commission #1:
  `services/commission-worker/worker.js:90` runs `JSON.parse` *outside* the
  try/catch (uncaught 500, not the intended fail-loud 502); there is **no Stripe
  idempotency** (at-least-once delivery + retries → duplicate GitHub issues per
  purchase); and the issue body is uncapped (`:47-63`), so a >65 536-char custom
  field makes GitHub reject the body forever, wedging the webhook on Stripe's
  retry loop. Gate C's "first paid commission" runs through this code.

## 4 · Full list

### Lane A — the shelf tells the truth

#### A1 · night-clerk catalog misdescribes two of nine plugins — FIX (lens: friendliness, helpfulness, fixes)
- **Evidence:** `plugins/night-clerk/data/catalog.json:31` (env-doctor "at
  session start" + `:34` `"hooks"` tag) vs `plugins/env-doctor/README.md` (no
  hooks); `:77` (pr-narrator "subagent") + `:79` `"agents"` tag vs the shipped
  skill at `plugins/pr-narrator/skills/pr/SKILL.md`. The clerk skill
  (`plugins/night-clerk/skills/clerk/SKILL.md`) recommends straight from this
  file, so the errors reach users as confident advice.
- **Proposal:** correct the two one-liners and two tags; regenerate via
  `tools/clerkcat.py`; bump night-clerk (Version law) so installed clerks get the
  fix. Add a `_tools` fixture asserting each catalog entry's `components`/tags
  match the plugin's actual manifest, so drift can't recur.
- **Why:** the marketplace's north star is "trust what loads." A discovery
  surface that invents a session-start hook is the exact opposite. **Impact H ·
  Effort S.**

#### A2 · env-doctor SessionStart hook — build what the catalog promises — NEW (lens: onboarding, helpfulness, feedback)
- **Evidence:** env-doctor ships only a manual `envcheck` skill
  (`plugins/env-doctor/skills/envcheck/SKILL.md`) and no hook; the catalog and its
  own value proposition ("warns early") imply an automatic check. The contradiction
  in A1 is best resolved *toward* the feature, not away from it.
- **Proposal:** a narrow, fail-open `SessionStart` hook that runs the toolchain
  check once on a fresh session and prints a one-line warning on mismatch —
  reviewed line-by-line under the hooks-are-guests law, with an env-var opt-out
  and a regression suite. Version bump + CHANGELOG.
- **Why:** the whole point of an env doctor is to catch you *before* you burn a
  session on the wrong Node/Python. This turns the quality category's only "quiet"
  plugin into one that actually fires. **Impact H · Effort M.**

#### A3 · commit-craft skill/hook config parity — FIX (lens: helpfulness, fixes)
- **Evidence:** `plugins/commit-craft/scripts/check-commit-msg.sh:41-43` reads
  `COMMIT_CRAFT_TYPES`; `plugins/commit-craft/skills/commit/SKILL.md:9` hardcodes
  `feat|fix|docs|refactor|test|chore|perf`. Customize the guard and the drafter
  drifts out of sync — drafts get rejected, or the guard allows types the skill
  never suggests.
- **Proposal:** have the skill reference the same env-var convention (or a single
  shared default), and add a suite case that a customized type list is honoured
  end to end. Version bump. (Also: `check-commit-msg.sh:20` uses the deprecated
  `datetime.utcnow()` — trivial to modernize while here.)
- **Why:** a configurable hook whose companion skill ignores the config is a trap
  the v10 "hook configurability" item opened without closing. **Impact M · Effort S.**

#### A4 · plugin-smith `doctor` description fails its own bar — FIX (lens: friendliness, fixes)
- **Evidence:** `plugins/plugin-smith/skills/doctor/SKILL.md:3` — *"Use when
  reviewing, debugging, or health-checking a plugin"* — is verb-only and generic;
  the same file (`:26`) instructs Claude to bounce descriptions that "couldn't
  tell Claude *when* to fire." Any generic "review this" request can shadow it.
- **Proposal:** rewrite with concrete trigger phrases ("audit my plugin against
  the spec," "why won't my hook load," "check plugin.json"), mirroring
  `skills/scaffold/SKILL.md:3`. Version bump.
- **Why:** the plugin that enforces description quality on the whole shelf should
  pass its own test — this is dogfooding in the literal sense. **Impact M ·
  Effort S.**

### Lane B — plugin depth & synergy

#### B5 · session-recap Stop-hook companion — NEW (lens: engagement, retention, helpfulness)

### Lane C — the gates that guard everything

#### C9 · `gates.yml` sitemap false-drift — FIX (lens: fixes, felt debt)
- **Evidence:** `.github/workflows/gates.yml`'s "generated output in sync" step
  reverts `site/data.json site/index.html site/feed.xml` before
  `git diff --exit-code`, but `tools/build.py:1298-1301` writes today's date into
  `site/sitemap.xml` `<lastmod>`. Every PR/push dated after the committed sitemap
  fails the gate for a reason unrelated to the change — the v9 flagship gate now
  cries wolf.
- **Proposal:** add `site/sitemap.xml` to the revert set, or make its `<lastmod>`
  derive from the record's real `updated` date instead of "now." Two-iteration ADR
  (touches `.github/` + `tools/`).
- **Why:** a gate that fails spuriously trains everyone to ignore it — the worst
  possible state for the check that guards mode:pr shifts. **Impact H · Effort S.**

#### C10 · commission-worker hardening — FIX (lens: fixes, reach/community)
- **Evidence:** `services/commission-worker/worker.js:90` — `JSON.parse(payload)`
  sits *before* the try that starts at `:94`, so a signature-valid non-JSON body is
  an uncaught 500 rather than the designed fail-loud 502. No idempotency: Stripe
  delivers `checkout.session.completed` at-least-once and retries non-2xx for days
  (`:100`), and every delivery calls `openIssue` (`:47-63`) → duplicate commission
  issues per purchase. The body is uncapped (`:47-63` embeds all of `request`),
  so a >65 536-char custom field is rejected by GitHub forever, wedging the webhook.
- **Proposal:** move `JSON.parse` inside the try; dedup on `session.id`
  (KV or an idempotency check before opening an issue); cap the request text
  length before embedding. Keep the strong parts (HMAC verify, 5-min replay window,
  constant-time compare, untrusted-text fencing).
- **Why:** Gate C's "first paid commission" is the first real revenue event; it
  should not double-bill the roadmap or brick the endpoint. **Impact M-H · Effort M.**

#### C11 · silent issue truncation in intake & metrics — FIX (lens: fixes, feedback)
- **Evidence:** `tools/intake.py:35` (`gh issue list`, default 30) and
  `tools/metrics.py:57-58` (`per_page=100`, no pagination) silently drop items past
  the cap — commissions/bugs/ideas beyond 30 never enter the backlog; votes and
  counts beyond 100 undercount the growth ledger the whole doctrine answers to.
- **Proposal:** explicit `--limit`/pagination on both; a `_tools` fixture with >30
  synthetic issues proving nothing is dropped.
- **Why:** the growth-honesty law forbids numbers the repo can't substantiate — a
  silently truncated count violates it by omission. **Impact M · Effort S.**

#### C12 · restamp crash guard + finish the front-matter dedup — FIX/IMPROVE (lens: fixes, felt debt)
- **Evidence:** `tools/restamp.py:36` does
  `re.search(r"^name: (.+)$", t, re.M).group(1)` with no None guard — a record

## 5 · Sequence — if only three ship first

1. **A1 — fix the catalog lies.** The marketplace's entire pitch is "trust what
