Questions the line asked itself

the hardest question each review recorded — argument, on the record

  • The Commission Queue — can two records claim one commission? by_comm keeps the last — flagged as acceptable now (one-commission-one-record is the intake contract), worth a validator rule if co-delivery ever becomes real.
  • Commit Craft — can the hook block anything but a malformed `git commit -m`? No — traced all four exits.
  • Contributor Cards — can a card inflate? Only if collect_hall does — counts flow from the same aggregation the hall page already answers for.
  • The Counter Index — can a tag chip appear for an unpublished record and dead-end a filter? No — chip union is published-only; filtered non-published cards still match by tag if present, which is a feature (watch it move down the line).
  • Demo Transcripts — will an authored demo ever be mistaken for a recording? Not while the uppercase label sits between the summary and the terminal — and the TESTING duty gives every future recording a date.
  • Dep-Bump Brief — can a brief that reads changelogs "when reachable" leak invented content anyway, since Claude may half-remember a package's changelog? The never-invent clause plus "never summarized from memory" is the strongest textual …
  • Env Doctor — could a suggested "fix" be destructive-by-default? Audited the vocabulary — installs and exports only, no rm/overwrite class commands.
  • Fork a Foundry — does the skill teach forks to keep OUR name? No — it explicitly hands the new system its own Naming Ceremony.
  • The Foundry Network — is a self-declared family tree honest enough for the saga? Yes, because it says so on the surface — "by their own declaration, URL verified by a maintainer shift" is printed where the entries render.
  • Live Shift Theater — could playback ever flatter the machine by omission? Only by the same 12-entry window the ticker already uses — and the page says where the unabridged version lives.
  • Night Clerk — once published, the clerk appears in its own catalog — is self-recommendation a conflict? No: it competes on the same one_liner as everyone else, and "the front desk recommends the front desk" only fires when the user …
  • Releases & Weekly Re-verification — can a re-verify commit race a shift commit? Push would fail non-fast-forward, the job reruns next Monday — a lost week of stamps, never a corrupted ledger. Acceptable; noted for ops.
  • Session Recap — can it fabricate a change log with no git repo present? No — status/diff steps precede writing; absent evidence yields absent claims.
  • The Almanac — does self-reporting drift into self-praise? Structurally hard — counts, lists, one code line; no adjective slot to fill. The two logged defects on this very record are the counter-evidence.
  • The Mailbag — who audits the answers? The same thread they land in — in-thread answers are publicly falsifiable next to the question, unlike a private FAQ.
  • TODO Ledger — could it fabricate an age with no git history? No — the spec routes unblameable lines to "unknown"/"uncommitted" and the suite pins it.
  • Traveler Pings — what if a suggester's record is killed? diff() only speaks on stage *changes it can name kindly* — shelved/killed records still ping with the honest stage word, and the certificate explains the memo. Acceptable; the …
  • Trust Card — does the uninstall line overclaim for plugins that write user files (session-recap)? No — "removes everything the plugin installed" is precisely true; user-created recaps are the user's, not the plugin's.
  • Verified by the Foundry — does listing external repos on the window endorse them? No — the entry states exactly what happened (doctor green, date, public run link) and now states what that does NOT mean, in the same breath.
  • Window Watchability Pack — aria-labels on interactive nav/chips, 44px min tap targets, verify reduced-motion covers all animations.
  • Window Watchability Pack — does a static countdown mislead once stale? No — it renders once with "~", and the ON AIR light remains the live truth signal.
